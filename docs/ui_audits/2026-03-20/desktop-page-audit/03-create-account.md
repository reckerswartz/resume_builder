# Create Account

## Scope

- **Route**: `/registration/new`
- **Audience**: New users
- **Primary goal**: Create an account and enter the workspace with confidence

## Strengths

- **Clear headline and action**: The page makes the onboarding intent easy to understand.
- **Useful product framing**: Explaining that a starter resume is created is a good onboarding cue.
- **Consistent form layout**: The form is clean and visually balanced on desktop.

## Findings

- **Medium - The page explains the starter experience, but not enough to reduce uncertainty**: It says a starter resume and template are created, but it does not show what that actually looks like or how editable it will be.
- **Medium - Password setup is under-supported**: There is no strength guidance, no minimum-expectation hint, no show/hide control, and no inline confirmation feedback until validation fails.
- **Medium - The support column repeats too much product marketing**: For a registration task, the left rail adds copy but not much conversion value. A more visual `what you get after signup` preview would work better.
- **Low - The sign-in handoff is present but understated**: Existing users can recover easily, but the page makes that choice feel secondary in a way that may frustrate returning visitors who land here by mistake.
- **Medium - Missing trust/compliance context**: There is no terms, privacy, or account-use note near the submit action. Even a lightweight statement would improve confidence.
- **Low - The page does not set expectations for the next screen**: Users are not told whether they will land in a dashboard, a builder, or an onboarding flow after signup.

## Recommended enhancements

- **Show the post-signup payoff**: Add a small starter workspace preview or template thumbnail instead of only describing it.
- **Upgrade password guidance**: Add visibility controls, requirements, and inline validation cues.
- **Add a minimal legal/trust note**: Keep it short, but make the account-creation commitment feel more polished and complete.
